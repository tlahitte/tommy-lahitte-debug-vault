  <action>
    Create `.github/workflows/scheduled-rebuild.yml`:

    ```yaml
    name: Scheduled Notion Rebuild
    on:
      schedule:
        - cron: '0 6 * * *'   # Daily at 06:00 UTC
      workflow_dispatch:         # Manual trigger button in GitHub UI

    jobs:
      rebuild:
        runs-on: ubuntu-latest
        steps:
          - name: POST to Netlify Build Hook
            run: curl -s -X POST "https://api.netlify.com/build_hooks/${TOKEN}"
            env:
              TOKEN: ${{ secrets.NETLIFY_BUILD_HOOK_TOKEN }}
    ```

    Important implementation notes:
    - Store only the alphanumeric token portion (after the last `/` in the Netlify hook URL) as the secret, not the full URL. The full URL is reconstructed in the `run:` command.
    - `workflow_dispatch` enables a "Run workflow" button in the GitHub Actions UI for manual triggers — required for testing without waiting for the cron.
    - The `schedule` cron runs daily at 06:00 UTC regardless of whether Notion content has changed. This is intentional — the cost is negligible and simpler than change detection.
    - Do NOT add any checkout step — the workflow only needs to fire the HTTP POST, no repo checkout required.
  </action>
