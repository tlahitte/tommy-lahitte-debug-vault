## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none — custom kraft token system | Phase 1 established |
| Preset | not applicable | no shadcn |
| Component library | none (manual components with Tailwind semantic tokens) | tailwind.config.ts |

**Note:** shadcn is not initialized and must NOT be initialized in this phase. The project uses a custom CSS custom property token layer bridged to Tailwind. All new components must use semantic tokens (`bg-surface`, `text-text-primary`, `text-text-muted`, `bg-accent`, `border-border`), never raw hex values or zinc/violet Tailwind classes.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline tight padding (e.g. `gap-1`) |
| sm | 8px | Compact element spacing, date/meta gaps (e.g. `gap-2`, `mb-2`) |
| md | 16px | Default element spacing, card padding, content gaps (e.g. `p-4`, `gap-4`) |
| lg | 24px | Section padding, article block rhythm (e.g. `p-6`, `space-y-6`) |
| xl | 32px | Layout gaps, article header margin (e.g. `mb-8`, `mt-8`) |
| 2xl | 48px | Major section breaks (e.g. `py-12`) |
| 3xl | 64px | Page-level bottom padding (e.g. `pb-16`, `pb-32`) |

Exceptions:
- Reading progress bar: 1–2px height (not a spacing token — it is a visual indicator dimension)
- Touch targets (interactive elements): minimum 44px height for accessibility
- Article content `space-y-4` (16px) between Notion blocks — matches existing rhythm from `[slug]/page.tsx`

|------|------|--------|-------------|---------------|--------|
| Body | 16px | 400 (regular) | 1.5 (`leading-relaxed`) | `text-base font-normal` | existing `[slug]/page.tsx` paragraph blocks |
| Label / Meta | 12px | 400 (regular) | 1.4 | `text-xs` | date, reading time, category labels |

- Maximum 2 weights: 400 (regular) and 600 (semibold) — `font-bold` (700) is not used

## Interaction Contracts

### Reading progress bar
- Behavior: appears at 0% on page load, fills left-to-right as user scrolls, reaches 100% at bottom of article content
- Position: `fixed top-0 left-0 w-full` — sits above the sticky header
- Dimensions: height `2px`, z-index above header
- Color: `bg-accent` (#C85A3A)
- Transition: no animation on the bar itself — CSS `width` updated via scroll listener (smooth inherently)
- Renders only on article detail page (`/blog/[slug]`), not on blog list page

### Blog list row hover
- Behavior: title color transitions from `text-text-primary` to `text-accent` on row hover
- Transition: `transition-colors duration-200`
- No lift/shadow on list rows (reserved for cards in Phase 4 — different component)
- "Read article" text appears in `text-accent`, no hover change needed (already accent)

### Article cover image
- Displayed if present: full-width, `aspect-video` (16:9), `object-cover`, `rounded-xl`
- No image: no placeholder shown in article detail (header flows directly to content)
- Images downloaded at build time to `/public/notion-images/` — served as static Next.js assets

### Notion block types — visual treatment

| Block type | Visual treatment |
|------------|-----------------|
| `paragraph` | `text-base text-text-primary leading-relaxed` |
| `heading_1` | `text-3xl font-semibold text-text-primary mt-10 mb-3` |
| `heading_2` | `text-lg font-semibold text-text-primary mt-8 mb-2` |
| `heading_3` | `text-base font-semibold text-text-primary mt-6 mb-1` |
| `code` | Existing CodeBlock component — `bg-zinc-900 text-zinc-200`, preserved dark interior |
| `callout` | `bg-surface-raised border border-border rounded-lg px-4 py-3 text-text-primary` with emoji icon if present |
| `bulleted_list` | `list-disc list-outside ml-5 space-y-1 text-text-primary` |
| `numbered_list` | `list-decimal list-outside ml-5 space-y-1 text-text-primary` |
| `image` | `w-full rounded-xl overflow-hidden` — caption in `text-xs text-text-muted text-center mt-2` |
| `toggle` | `<details>` / `<summary>` — summary styled `font-medium text-text-primary cursor-pointer`, body indented `ml-4 mt-2` |
| `divider` | `<hr class="border-t border-border my-8">` |
| unsupported | Silent fallback — renders nothing, no build error (NOTI-03) |

### Page transitions
- Inherited from Phase 1 `template.tsx` — `AnimatePresence` with 250ms duration
- No additional motion work needed in Phase 2

## Copywriting Contract

| Element | Copy | Source |
|---------|------|--------|
| Blog list page heading | "Blog" | existing blog/page.tsx — preserved |
| Blog list page subheading | "Things I am currently working on." | existing — preserved |
| Article back link | "← Back to Blog" | existing `[slug]/page.tsx` — preserved |
| Reading time label | "{N} min read" | new — adjacent to date in article header |
| "Read article" list row CTA | "Read article" | new inline text link on list rows |
| External link CTA (if post.link present) | "{post.link.label} →" | existing — preserved, driven by Notion data |
| Empty state heading | "No articles yet." | for when Notion DB returns 0 published posts |
| Empty state body | "Check back soon — something is in progress." | reassuring, no next-step action needed (read-only site) |
| Error state (article not found) | handled by Next.js `notFound()` — renders existing `not-found.tsx` page | existing — no new copy needed |
| Notion API error at build | Build fails loudly — no user-facing error state. Static export: if build fails, old version stays live. | architecture decision, not UI copy |

**No destructive actions exist in Phase 2.** The site is read-only for visitors; content management is in Notion.

## Phase-Specific Notes

### No dark mode
Dark mode is explicitly out of scope (REQUIREMENTS.md). All color values are light palette only.

### No filter UI
Blog category/tag filtering is deferred to v2 (BLOG-V2-02). The list page has no filter controls in Phase 2.

### Static export constraint
`output: 'export'` is in effect. No server components that make runtime API calls. All Notion API calls happen at build time only. No `use server` or Server Actions in blog routes.

### SEO — slug preservation
`generateStaticParams` must return all Notion posts. `generateMetadata` must produce canonical URLs. Slugs `elevenapps-immersive-show-swift` and `technical-director-people-person-show` must resolve exactly.
