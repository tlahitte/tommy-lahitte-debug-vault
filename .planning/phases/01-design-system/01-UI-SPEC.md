## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none — manual Tailwind token layer | RESEARCH.md: no shadcn, pure Tailwind v3 |
| Preset | not applicable | — |
| Component library | none — custom components | RESEARCH.md |
| Icon library | none declared for Phase 1 | default |

**Token architecture:** Two-layer. CSS custom properties defined in `globals.css :root`, bridged into `tailwind.config.ts` via `var(--token)` references. Semantic token names used exclusively in all component class strings — no raw zinc/violet Tailwind classes permitted after migration.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tag inline padding, metadata gaps |
| sm | 8px | Compact element spacing, label-to-input gaps |
| md | 16px | Default element spacing, card padding inner |
| lg | 24px | Section padding, card content padding |
| xl | 32px | Layout column gaps, section vertical rhythm |
| 2xl | 48px | Major section breaks, above/below headings |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:
- Navigation height: 56px (h-14 in Tailwind = 3.5rem). Non-standard but intentional — matches locked Header decision.
- Scroll threshold for header opacity transition: 48px (`window.scrollY > 48`). Claude's discretion per CONTEXT.md.
- Focus ring offset: 2px. Accessibility standard, below xs.

## Color

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `surface` / `background` | `#FAFAF7` | Page background, body, transparent header state |
| Secondary (30%) | `surface-raised` | `#F0EBE0` | Cards, panels, code block outer, nav opaque state |
| Text primary | `text-primary` / `foreground` | `#1A1816` | Body text, headings, nav brand name |
| Text muted | `text-muted` | `#7A6F68` | Dates, captions, nav links (default state), placeholder text |
| Accent (10%) | `accent` | `#C85A3A` | Accent reserved list (see below) |
| Accent hover | `accent-hover` | `#A8432A` | Hover state of any accent element (~20% darker) |
| Border | `border` | `#E2D9CE` | Card borders, dividers, header border-bottom on scroll, tag borders |

**Accent (`#C85A3A`) is reserved exclusively for:**
1. Active/hover nav link text color (`hover:text-accent`)
2. Blog post title hover color
3. "Read more" link text in BlogCard
4. Tag component background on hover
5. Focus ring outline (accessibility coherence with industrial palette)
6. Card hover box-shadow tint: `rgba(200, 90, 58, 0.12)`
7. CategoryBadge glow update (replacing violet badge glow with accent-adjacent warm palette)

Destructive: not applicable in Phase 1 — no destructive user actions exist in the design system migration phase.

Dark mode: out of scope (REQUIREMENTS.md: "Thème sombre — Out of Scope").

## Interaction Contract

### Header scroll behavior
- At `scrollY === 0`: `bg-transparent border-b border-transparent` — fully transparent
- At `scrollY > 48px`: `bg-surface border-b border-border` — opaque, with bottom border
- Transition: `transition-all duration-300` (300ms, Tailwind default ease)
- Implementation: `useState(false)` + `useEffect` scroll listener with `{ passive: true }`. No library needed.

### Card hover (BlogCard, future project cards)
- Transform: `translateY(-2px)` — "physical object lifts off surface"
- Shadow: `0 8px 24px -4px rgba(200, 90, 58, 0.12)` — warm rust tint, subtle
- Duration: 200ms, `ease: 'easeOut'`
- Implementation: `motion.div` with `whileHover={{ y: -2, boxShadow: '...' }}` from `motion/react`

### Nav link hover
- Color transition: `text-muted` → `text-accent` (`#7A6F68` → `#C85A3A`)
- Duration: `transition-colors` (150ms Tailwind default)
- Implementation: Tailwind `hover:text-accent transition-colors`

### Page transitions
- Entry animation: `opacity: 0, y: 8` → `opacity: 1, y: 0`
- Duration: 250ms, `ease: 'easeOut'`
- Implementation: `src/app/template.tsx` with `"use client"` + `motion.div` from `motion/react`
- AnimatePresence: NOT used for cross-route exit animations (fragile in App Router) — entry-only via `template.tsx`

### Focus states
- Outline: `2px solid #C85A3A` (accent color) with `outline-offset: 2px`
- Applied to: all interactive elements (links, buttons, inputs)
- Tailwind: `focus-visible:outline-2 focus-visible:outline-accent focus-visible:outline-offset-2`

### Animation library
- Package: `motion` (formerly `framer-motion` — rebranded, same API)
- Import: `import { motion, AnimatePresence } from 'motion/react'`
- Do NOT install or import from `framer-motion` package — causes duplicate bundle weight
