**Create `src/components/blog/StatusBadge.tsx`:**

A server component (no `'use client'`) that renders a pill-shaped badge with color coding based on status value. Uses kraft semantic tokens.

```typescript
import type { ProjectStatus } from '@/lib/blog-types'

const STATUS_STYLES: Record<ProjectStatus, string> = {
  'In Progress': 'bg-amber-100 text-amber-800 border-amber-300',
  'Complete': 'bg-emerald-100 text-emerald-800 border-emerald-300',
  'Archived': 'bg-zinc-100 text-zinc-600 border-zinc-300',
}

interface StatusBadgeProps {
  status: ProjectStatus
}

export default function StatusBadge({ status }: StatusBadgeProps) {
  return (
    <span
      className={`inline-flex items-center text-xs font-semibold px-2.5 py-1 rounded-full border ${STATUS_STYLES[status]}`}
    >
      {status}
    </span>
  )
}
```

Color rationale:
- In Progress: amber (warm, active — matches kraft warm palette feel)
- Complete: emerald (positive/done)
- Archived: zinc (neutral/faded)
These are utility colors for status semantics, not brand colors — using Tailwind built-in colors is intentional here (same convention as CategoryBadge using blue/red/green for categories in the original design before Phase 1 replaced them).

**Create `src/components/blog/ProjectGallery.tsx`:**

A server component rendering a responsive CSS grid of downloaded gallery images. Placed between header and body on project detail pages.

```typescript
interface ProjectGalleryProps {
  images: string[]
}

export default function ProjectGallery({ images }: ProjectGalleryProps) {
  if (images.length === 0) return null

  return (
    <section className="my-10" aria-label="Project gallery">
      <div className="grid grid-cols-2 sm:grid-cols-3 gap-3">
        {images.map((src, i) => (
          // eslint-disable-next-line @next/next/no-img-element
          <img
            key={i}
            src={src}
            alt={`Project photo ${i + 1}`}
            className="w-full rounded-lg aspect-square object-cover border border-border"
            loading="lazy"
          />
        ))}
      </div>
    </section>
  )
}
```

Gallery design decisions:
- CSS grid: 2 columns on mobile, 3 on sm+ breakpoint
- aspect-square with object-cover for uniform thumbnails
- border-border for subtle kraft-consistent framing
- No lightbox — simple grid fits static export and kraft identity
- loading="lazy" for performance on image-heavy project pages
- aria-label for accessibility
